# 🍽️ 食事中のワークフローガイド

食事中に素早くデモを作成するための実践的なガイドです。

## 📋 事前準備（食事前）

- [x] プロジェクトのセットアップ完了
- [ ] VSCodeでプロジェクトを開く
- [ ] 開発サーバーを起動: `npm run dev`
- [ ] ブラウザで http://localhost:3000 を開く
- [ ] QUICK_REFERENCE.mdをブラウザで開いておく

## 🎯 食事中のステップ

### Step 1: ヒアリング（5-10分）
取引先の話を聞きながら、メモを取ります。

**聞くべきポイント:**
- 何を解決したいのか？（課題）
- 誰が使うのか？（ターゲット）
- どんな機能が必要か？（要件）
- 優先順位は？

**即座にメモ:**
```
REQUIREMENTS.mdに追記してください:

【概要】
[取引先名]向けの[システム名]

【目的】
- [解決したい課題]

【主な機能】
1. [機能1]
2. [機能2]
3. [機能3]

【優先順位】
1. [最優先機能]
2. [次に重要]
```

### Step 2: AIエージェントに指示（3-5分）

**型定義を作成:**
```
REQUIREMENTS.mdの内容を読み取って、
src/types/index.tsに必要な型定義を追加してください。
```

**ダミーデータを生成:**
```
定義した型に基づいて、
src/data/sampleData.tsに20件程度のダミーデータを生成してください。
```

### Step 3: UI構築（10-20分）

**メインページの作成:**
```
以下の画面を作成してください:

1. [画面名] - [説明]
   - [表示要素1]
   - [表示要素2]
   - [機能]

src/app/page.tsxを編集して実装してください。
```

**追加ページの作成（必要なら）:**
```
[ページ名]ページを作成してください:
- パス: /[path]
- 機能: [説明]
- 表示データ: [データ種類]
```

### Step 4: デモ確認（5分）
ブラウザで動作を確認しながら、取引先に見せる

**チェックポイント:**
- [ ] データが正しく表示されている
- [ ] レイアウトが崩れていない
- [ ] モバイルでも見やすい
- [ ] 操作が直感的

### Step 5: フィードバック反映（残り時間）
取引先のフィードバックを即座に反映

```
[フィードバック内容]を反映してください。
例: 
- 「もっと大きく表示してほしい」
- 「色を変えてほしい」
- 「この情報も表示したい」
```

## 🎯 時間配分の目安

**60分の食事の場合:**
- 10分: ヒアリング
- 5分: 型定義・データ作成
- 25分: UI構築
- 10分: デモ確認・議論
- 10分: フィードバック反映

**90分の食事の場合:**
- 15分: ヒアリング
- 10分: 型定義・データ作成
- 35分: UI構築
- 15分: デモ確認・議論
- 15分: フィードバック反映

## 🚨 トラブルシューティング

### ビルドエラーが出たら
```bash
# エラーメッセージをコピーしてAIエージェントに
「以下のエラーを修正してください: [エラーメッセージ]」
```

### データが表示されない
```bash
# ブラウザのコンソールを確認
F12 > Console タブ
```

### スタイルが崩れた
```bash
# キャッシュをクリア
Cmd + Shift + R (Mac)
Ctrl + Shift + R (Windows)
```

## 📱 デモのコツ

1. **ストーリーを語る**: 「こんな感じで使えます」
2. **操作を見せる**: クリックやスクロールを実演
3. **データの意味を説明**: 「これは顧客情報で...」
4. **拡張性をアピール**: 「ここはもっと拡張できます」
